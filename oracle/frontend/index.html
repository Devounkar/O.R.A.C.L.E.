How It Works

ORACLE analyses accuracy per tier against token cost. If a tier's accuracy is below 70%, beam width increases. If above 92%, it decreases to save compute. Excessively long responses trigger a temperature reduction to focus the generator.


Rules:
• accuracy < 0.70 → beam_width + 2
• accuracy > 0.92 → beam_width − 1
• avg_tokens > 1500 → temperature − 0.1

System Architecture

        
ORACLE — Outcome-Ranked Adaptive Cognitive Logic Engine

      

      

        

          
01 · Difficulty Estimator

          
Heuristic feature extraction: word count, clause depth, negation count, multi-hop signal density, named entity count. Assigns a raw score mapped to Easy / Medium / Hard tiers. O(1), no API call.

        

        

          
02 · Strategy Selector

          
Looks up a (difficulty × question_type) table to retrieve beam_width, temperature, CoT depth, and verification flags. Supports live override via the recalibration loop.

        

        

          
03 · CoT Generator

          
Fires k independent API samples with slightly varying temperature for diversity. Parses each into a (chain_of_thought, final_answer) pair. Assigns a proxy logprob via output-length normalisation.

        

        

          
04 · Verifier Model

          
Separate LLM call evaluating (question, CoT, answer) on logical coherence, faithfulness, plausibility, and completeness. Returns a 0–1 JSON score. Acts as a process reward model.

        

        

          
05 · Self-Consistency Aggregator

          
Clusters answers by normalised string equality (or numeric tolerance). Each cluster's weight = Σ composite_scores. Winning cluster's best path is returned.

        

        

          
06 · Benchmark Logger

          
Appends every QueryRecord to a JSONL file. Computes accuracy, avg tokens, latency per tier. Feeds the recalibration loop that adjusts the strategy table dynamically.
